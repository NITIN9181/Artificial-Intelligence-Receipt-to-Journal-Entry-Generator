# Contributing to AI Receipt to Journal Entry Generator

First off, thank you for considering contributing to this project! It's people like you who make this tool better for everyone.

## 🌈 How Can I Contribute?

### Reporting Bugs
- Check the [Issues](https://github.com/yourusername/ai-receipt-journal/issues) to see if the bug has already been reported.
- If not, open a new issue. Include a clear title, a description of the problem, and steps to reproduce.

### Suggesting Enhancements
- Open an issue with the "enhancement" label.
- Describe the feature you'd like to see and why it would be useful.

### Pull Requests
1. **Fork** the repository.
2. **Create a branch** for your changes: `git checkout -b feature/your-feature-name` or `fix/your-bug-name`.
3. **Make your changes**.
4. **Run tests**: Ensure all backend and frontend tests pass.
   - Backend: `pytest`
   - Frontend: `npm test`
5. **Commit your changes**: Use [Conventional Commits](https://www.conventionalcommits.org/).
6. **Push to your fork**: `git push origin your-branch-name`.
7. **Open a Pull Request**.

## 💻 Development Setup

See the [README.md](README.md#%EF%B8%8F-installation) for detailed setup instructions for the backend and frontend.

### Python Style Guide (Backend)
- Use `black` for formatting.
- Use `ruff` for linting.
- Add docstrings to all public functions and classes.
- Ensure 90%+ test coverage.

### TypeScript Style Guide (Frontend)
- Use `eslint` and `prettier`.
- Use TypeScript strict mode.
- Prefer functional components and hooks.

## 📜 Code of Conduct
Please be respectful and professional in all interactions within the project. We aim to foster an inclusive and welcoming environment.
